# Work-Item And SOP Skeleton

This skeleton shows how to describe a work-item-like operating procedure
without pretending that the public example is an official SOP.

## Minimum Sections

1. Purpose
2. Scope
3. Roles
4. Preconditions
5. Inputs
6. Procedure
7. Expected evidence
8. Review and approval boundary
9. Trace and handoff boundary
10. Change history

## Procedure Style

Write the procedure so another qualified person can reproduce the work:

- arrival screen or artifact location;
- operation to perform;
- expected confirmation result;
- evidence to keep;
- exception path when the result is not available.

## Boundary

A skeleton is not a controlled SOP. It becomes a controlled SOP only when the
target organization reviews, owns, versions, approves, and stores it in the
formal system.

Use the No-X rule:

```text
NO-CANDIDATE-AS-RECORD
```
