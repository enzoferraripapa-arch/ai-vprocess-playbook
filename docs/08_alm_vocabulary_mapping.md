# ALM Vocabulary Mapping

This playbook uses vendor-neutral words where possible. Different ALM tools and
organizations may use different labels for the same operating concept.

| Playbook term | Typical meaning |
| --- | --- |
| Work item | A controlled unit of work, requirement, issue, test, risk, change, or review record. |
| Field | A structured attribute on a work item. |
| Type | The category or class of a work item. |
| Link role | A typed relationship between two records. |
| Workflow state | The lifecycle status of a record. |
| Baseline | A controlled snapshot or approved reference point. |
| Review gate | A decision point where evidence is checked before progress. |
| Handoff package | A reviewed local package prepared for manual entry or approved import. |

## Rule

Map the terms to your tool and organization before using the playbook. If the
tool label and the project rule differ, follow the approved project rule and
record the mapping.
