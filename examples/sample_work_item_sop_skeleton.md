# Sample Work-Item SOP Skeleton

Status: fictional example only.

## Purpose

Create or update a work-item-like record while preserving source, rationale,
review, and trace context.

## Scope

Use this skeleton for projects that need reviewable records before formal
system entry. Adapt it to the target organization's approved process.

## Preconditions

- The project profile is known.
- The record type and owner are known.
- The source evidence is available or the missing evidence is recorded.
- The reviewer boundary is defined.

## Procedure

| Step | Operation | Confirmation | Evidence |
| --- | --- | --- | --- |
| WI-01 | Identify the source and intended record type. | Source and type are visible. | Source id or source note. |
| WI-02 | Draft the record title, description, and rationale. | Draft is marked as candidate. | Candidate record text. |
| WI-03 | Add trace candidates. | Source and target are both explicit. | Trace candidate list. |
| WI-04 | Run the review gate. | Reviewer accepts, rejects, or requests changes. | Review disposition. |
| WI-05 | Prepare handoff package if accepted locally. | Package checksum and target scope are recorded. | Handoff package candidate. |

## Boundary

This skeleton is not a controlled SOP and the drafted record is not an official
record. Apply `NO-CANDIDATE-AS-RECORD` and `NO-HANDOFF-AS-IMPORT`.
