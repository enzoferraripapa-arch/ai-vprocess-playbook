# No-X Rules

Adopt these rules before using AI-generated engineering output in a project.

| Rule | Boundary |
| --- | --- |
| `NO-ROUTING-AS-APPROVAL` | A recommended activity route is not approval to proceed. |
| `NO-CANDIDATE-AS-RECORD` | A candidate artifact is not a controlled record. |
| `NO-GATE-CANDIDATE-AS-PASS` | A proposed gate outcome is not a passed gate. |
| `NO-TRACE-CANDIDATE-AS-TRACEABILITY` | A suggested trace is not accepted traceability. |
| `NO-HANDOFF-AS-IMPORT` | A handoff package is not a formal-system import. |
| `NO-LOCAL-ACCEPTED-AS-FORMAL-APPROVAL` | Local acceptance is not formal ALM/QMS approval. |
| `NO-PACK-UPDATE-AS-PROJECT-APPROVAL` | Updating common knowledge does not approve project decisions. |
| `NO-APP-AS-AUTHORITY` | A skill, app, or dashboard is not the authority. |

If a section has no evidence yet, write that directly. Do not leave a polite
empty placeholder that looks like a record exists.
